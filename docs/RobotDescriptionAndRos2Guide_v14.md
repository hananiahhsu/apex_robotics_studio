# Robot Description and ROS2 / MoveIt Guide v1.4

## Importing a Xacro with package discovery

The importer now resolves package locations through:

- `UrdfImportOptions::PackageSearchRoots`
- `AMENT_PREFIX_PATH`
- `COLCON_PREFIX_PATH`
- `ROS_PACKAGE_PATH`
- direct `package.xml` discovery under the candidate roots

Supported ROS-style substitutions in the importer include:

- `$(find package_name)`
- `$(env VAR)`
- `$(optenv VAR default)`
- `$(dirname)`

Example:

```bash
export ROS_PACKAGE_PATH=$PWD/samples/xacro_ws/src
export APEX_DEMO_ROBOT_NAME=ApexPainterV14
./out/build/v14dbg/apex_robotics_studio import-robot-description \
  samples/xacro/demo_painter_cell_v14.xacro \
  samples/projects/imported_from_xacro_v14.arsproject
```

Artifacts generated by the command:

- `samples/projects/imported_from_xacro_v14.arsproject`
- `samples/projects/imported_from_xacro_v14.expanded.urdf`

## Exporting a ROS2 / MoveIt workspace

```bash
./out/build/v14dbg/apex_robotics_studio export-ros2-workspace \
  samples/projects/imported_from_xacro_v14.arsproject \
  samples/ros2/generated_v14
```

Generated package layout:

- `apexpainterv14_imported_project_description`
- `apexpainterv14_imported_project_bringup`
- `apexpainterv14_imported_project_moveit_config`

Notable generated files:

- description package URDF with rewritten mesh URIs
- bringup bridge node
- MoveIt smoke test
- generated planning-scene loader node
- `config/scene_objects.yaml`
- `rviz/moveit_demo.rviz`

## Why the scene loader matters

The generated scene loader gives the exported MoveIt package a concrete handoff path for workcell obstacle geometry. It is not a full production planning-scene monitor, but it is a realistic bootstrap artifact for continuing into a real ROS2 / MoveIt workspace.
